1/ Introduction

La reconnaissance automatique de la parole est un domaine important de la science des données. Elle est utilisée dans de nombreuses applications comme les assistants vocaux ou la transcription automatique. Son objectif est d’analyser des signaux audio afin d’identifier des unités du langage appelées phonèmes.

Dans ce projet, nous utilisons un jeu de données issu de The Elements of Statistical Learning, construit à partir de la base TIMIT. L’objectif est de reconnaître cinq phonèmes différents à partir de caractéristiques extraites de signaux vocaux.

Il s’agit d’un problème de classification supervisée avec un grand nombre de variables explicatives, souvent corrélées. Pour cela, nous utilisons des méthodes adaptées comme la PLS-DA et l’analyse discriminante.

L’étude se déroule en deux étapes. D’abord, une analyse exploratoire permet de mieux comprendre les données.Ensuite, plusieurs méthodes de classification supervisée sont mises en place et comparées entre elles sur l’échantillon d’apprentissage. Enfin à l’issue de cette comparaison, seul le modèle le plus performant est retenu. Celui-ci est ensuite évalué sur un échantillon test afin de mesurer sa capacité de généralisation.

2/ Présentation du jeu de données

Code 
# IMPORT DES DONNEES

str(phoneme)

Le jeu de données est constitué de 4509 observations correspondant à des trames vocales de 32 millisecondes extraites de discours. Chaque trame est décrite par 256 variables numériques (x.1 à x.256) issues du log-périodogramme du signal audio.

La variable à expliquer est g, qui correspond au phonème prononcé et comporte cinq modalités : “sh”, “dcl”, “iy”, “aa” et “ao”. La colonne speaker identifie le locuteur et contient des informations supplémentaires, notamment le type de trame (train ou test), le code du locuteur, le code de la phrase prononcée et le dialecte. Cette variable sera utilisée pour distinguer les échantillons d’apprentissage et de test.

3/ Analyse exploratoire non supervisée

3.1 ACP (Analyse en Composantes Principales)

3.1.1 Justification de l’Analyse en Composantes Principales

Nous passons maintenant à la partie d’analyse factorielle. Nous choisissons de réaliser une analyse en composantes principales (ACP), car notre jeu de données est composé principalement de variables quantitatives (x.1 à x.256). L’ACP simplifie l’analyse en réduisant le nombre de variables sans perte significative d’information, ce qui est particulièrement utile ici, étant donné le grand nombre de variables corrélées entre elles.

Cette analyse permettra de mieux comprendre la structure des données et d’observer si certaines trames vocales (ou phonèmes) présentent des regroupements naturels sur la base de leurs caractéristiques spectrales.

Pour s’assurer que l’ACP est pertinente :

3.1.1.1 Indice KMO

L’indice KMO permet d’évaluer si les corrélations entre variables sont suffisantes pour être synthétisées par une analyse factorielle.

Code 
kmo_result <- KMO(X_train)
kmo_result$MSA  
[1] 0.9953065

Le résultat de l’indice KMO est supérieur à 0,6, il est exactement de 0,995, indiquant une adéquation très bonne, il y a des corrélations suffisantes entre certaines variables pour justifier une ACP ainsi l’analyse factorielle est pertinente.

3.1.1.2 Shapiro-Wilk

Code 
p_values <- sapply(X_train, function(x) shapiro.test(x)$p.value)
nb_var_pas_normales <- sum(p_values < 0.05)# Combien de variables ne sont pas normales
cat("Nombre de variables qui ne suivent pas une loi normale :", nb_var_pas_normales, "\n")
Nombre de variables qui ne suivent pas une loi normale : 256

Le test de normalité de Shapiro-Wilk indique que toutes nos variables explicatives ne suivent pas une loi normale, ce qui est attendu pour des données spectrale. Etant donné l’asymétrie des données, le test de bartlett n’est alors pas fiable. On doit donc utiliser une autre méthode pour vérifier la corrélation entre les variables.

3.1.1.3 Déterminant de la matrice de corrélation

Code 
det_corr <- det(cor(X_train))
det_corr
[1] 2.811738e-212

Le déterminant est très proche de 0 indiquant que certaines variables sont très fortement corrélées, ce qui renforce l’intérêt de l’ACP pour réduire la dimension et synthétiser l’information.

Ensuite, nous réalisons une ACP sur l’ensemble des 256 variables, sans suppression initiale, afin d’identifier les composantes principales et les regroupements naturels des phonèmes sur le plan factoriel.

Dans cette ACP, nous avons choisi de standardiser les variables (scale.unit = TRUE). Cela signifie que chaque variable est centrée et réduite, c’est-à-dire que sa moyenne devient 0 et son écart-type 1.

3.1.2 ACP sur l’ensemble des varaibles explicatives

3.1.2.1 Standardisation des variables

Code 
# Réalisation de l'ACP sur X_train

Cette étape est importante car certaines variables spectrales peuvent avoir des valeurs ou des variances plus élevées que d’autres. Sans standardisation, les variables les plus “grandes” domineraient l’analyse et fausseraient les composantes principales. La standardisation permet ainsi que toutes les variables contribuent équitablement, ce qui est essentiel pour interpréter correctement les regroupements naturels des phonèmes.

Cependant, ces résultats sont à interpréter avec prudence compte tenu de la forte multicolinéarité des variables vu au moment du calcul du déterminant, mais confirment l’intérêt des méthodes de réduction de dimension.

4/ Apprentissage supervisé

L’analyse exploratoire non supervisée (ACP) a permis de visualiser la structure générale des données et d’identifier des regroupements naturels entre les phonèmes. Ces informations servent de repères pour guider l’analyse supervisée, mais elles ne permettent pas à elles seules de classer les phonèmes.

Nous passons donc maintenant à la PLS-DA (Partial Least Squares Discriminant Analysis), une méthode supervisée adaptée aux données de grande dimension et fortement corrélées. Elle permet de réduire la dimension tout en maximisant la séparation entre les cinq phonèmes : “aa”, “ao”, “dcl”, “iy” et “sh”. Contrairement à la PLS classique, qui s’applique uniquement aux variables quantitatives, la PLS-DA utilise l’information des classes pour construire des axes discriminants efficaces pour la classification des données audio.

4.1 PLS-DA

On entraîne un modèle PLS-DA avec 10 composantes sur l’ensemble des données. Nous en prenons 10 car nous ne savons pas encore combien de composantes sont vraiment utiles, on affinera le nombre de composantes par la suite. On choisit également ce nombre car dans la littérature, nous avons vu qu’il était très rare de prendre au delà de 10 composantes donc nous supposons que c’est également le cas ici.

Le modèle PLS-DA a donc été ajusté sur le jeu de données composé de 3340 observations et 256 variables explicatives, avec une variable Y comportant 5 classes correspondant aux phonèmes. Dans un premier temps, il a été évalué sur l’ensemble des composantes et sans sélection préalable des variables.

Avant de passer à la description de ce graphe, on notera que overall représente le taux d’erreur global moyen pour l’ensemble des classes; BER la moyenne des erreurs par classe tandis que les trois types de point représentent chacun une manière d’assignation

L’analyse de la performance du modèle montre une diminution du taux d’erreur entre la 1ère et la 5ème composante, passe d’environ 40% à moins de 10%. Entre les composantes 3 et 5, l’erreur se stabilise. Au delà le gain de classification devient négligeable. A partir de 5, les courbes sont presques toutes stables, indiquant un risque de surapprentissage si on sélectionne plus de dimensions.

On observe également que les distances “max.dist” (rose) et “mahalanobis.dist” (noir) offrent les meilleures performances finales, De plus, on a un taux d’erreur global (overall) et un taux d’erreur équilibré (BER - Balanced Error Rate) très proches.

Le premier mesurant la proportion d’individus mal classés dans les données et le deuxième le taux d’erreur pour chaque classe. Leur proximité indique donc que le modèle a des performances similaires pour chaque phonème et qu’il n’y a pas de biais d’estimation. Ainsi, les conclusions qu’on peut tirer sont comparables entre chaque groupe. Cependant, la “centroids.dist” est beaucoup plus haute que les deux autres, montrant son inadéquation pour ce jeu de données. A choisir, on optera pour une des deux premières. Soit une assignation selon le score latent (max.dist) soit en tenant compte de la variance et de la corrélation (Mahalanobis.dist).

L’erreur résiduelle inférieure à 10% est par ailleurs cohérente avec le chevauchement partiel observé entre “aa”, “ao” et “iy” dans le plan des individus. Elle reflète une proximité acoustique réelle plutôt qu’une limite du modèle.

7/ Synthèse finale

L’objectif de ce projet était de comparer différentes méthodes de classification afin de discriminer cinq phonèmes à partir de données spectrales de grande dimension.

L’analyse exploratoire par ACP a permis de mettre en évidence la structure des données et les proximités entre certains phonèmes, notamment entre “aa” et “ao”, qui apparaissent difficilement séparables.

Plusieurs modèles supervisés ont ensuite été mis en œuvre : PLS-DA, PLS-DA réduit, sPLS-DA et AFD. Les résultats montrent que l’ensemble des méthodes offre de bonnes performances de classification, avec des taux d’erreur relativement faibles et des AUC élevées.

La PLS-DA complète fournit de bonnes performances mais reste peu interprétable en raison du grand nombre de variables. Les versions avec sélection de variables (PLS-DA réduit et sPLS-DA) permettent de simplifier le modèle tout en conservant des performances proches, ce qui améliore la lisibilité et limite le risque de surapprentissage.

L’AFD se distingue par des meilleures performances globales, avec le taux d’erreur le plus faible. Toutefois, certaines hypothèses de ce modèle ne sont pas pleinement respectées dans nos données, notamment la normalité des variables et l’homogénéité des matrices de covariance (hétéroscédasticité). Dans ce contexte, une analyse discriminante quadratique (QDA) serait théoriquement plus appropriée. Néanmoins, ne disposant pas des connaissances nécessaires pour l’implémenter correctement, cette approche n’a pas été explorée dans ce rapport. Elle constituerait cependant une piste intéressante afin de confirmer la supériorité du modèle retenu.

Au regard des résultats, les deux modèles sont pertinent dans ce contexte. Toutefois, la sPLS-DA reste plus adaptée aux données de grande dimension, notamment lorsque la sélection de variables est un enjeu important.

Enfin, l’ensemble des modèles met en évidence une difficulté persistante à distinguer certains phonèmes proches, ce qui reflète davantage une réalité linguistique qu’une limite des méthodes utilisées.
